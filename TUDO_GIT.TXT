Palavras-chave: Status, criar, adicionar, salvar, enviar, sincronizar, clonar, remover, log, mover, renomear, vim, desfazer, ignorar;


CONCEITOS IMPORTANTES

Changes to be committed
Arquivos que já passaram pelo git add e vão entrar no próximo commit.
Essas versões dos arquivos serão enviadas ao GitHub quando você der git push (depois do commit).

Changes not staged for commit
Os arquivos foram modificados, mas ainda não passaram pelo git add.
Não entram no commit se você commitar agora.
As modificações existem só no seu computador e não serão enviadas ao GitHub no próximo push.

Untracked files
Arquivos novos, que o Git ainda não conhece.
Só entram no controle de versão se você usar git add.
Enquanto isso, o Git ignora esses arquivos.




COMANDOS GIT

- git status (mostra o status dos git) PC: Status
As mudanças do projeto podem ser verificadas com o git status;
Este comando é utilizado muito frequentemente;
Aqui são mapeadas todas as alterações do projeto;
Como arquivos não monitorados e arquivos modificados;
Podemos também dizer que é a diferença do que já está enviado ao servidor ou projeto salvo.


- git init (cria o arquivo .git) -PC: criar


- git add (adiciona o arquivo no git) - “seleciona” -PC: adicionar
Podemos adicionar um arquivo específico ou diversos de uma vez só;
Somente adicionando arquivos eles serão monitorados pelo git;
Ou seja, se não for adicionar ele não estará no controle de versão;
É interessante utilizar este comando de tempos em tempos para não perder algo por descuido.


- git commit -m (mensagem) - “salva” -PC: salvar
             -a (Pula o git add - apenas para arquivos que ja estao versionados)
As alterações salvas do projeto são realizadas por: git commit;
Podemos commitar arquivo específicos ou varios de uma vez com a flag -a;
É uma boa prática enviar umar mensagem a cada commit, com as alterações que foram feitas


- git push -u (atualiza o repositório) “envia” -PC: enviar
Quando finalizamos uma funcionalidade nova, enviamos o codigo ao repositório remoto, que é código-fonte;
Esta ação e feita pelo git push;
Apos esta ação, o código do servidor será atualizado baseando-se no código local enviado;


- git pull (sincorniza o repositório remoto com o local) “sincroniza” -PC: sincronizar
É comum tambem ter que sicronizar o local com as mudanças do remoto, esta ação é feita pelo git pull.
Apos o comando, serão buscada atualizações, se econtradas elas serão unidas ao código atual existente na nossa maquina.
Você pode baixar novos arquivos e alterações feitas por outros desenvolvedores.


-git clone (Colocar o link do repositório remoto na frente.) PC: clonar
O ato de baixar um repositório remoto para o seu computador é chamado de clonar repositório;
Passando a referencia do repositório remoto, o git clone baixa todo o conteúdo do repositório para o seu computador;    
Este comando é utilizado quando entramos em um novo porjeto, por exemplo.

- git branch -M main (cria a branch principal)

- git remote add origin (origem do repositório)

- git remote rm origin (remove a origin)

- git remote -v (mostra a origin atual)


-git rm (nome do arquivo) PC: remover
Os arquivos podem ser deletados da monitoração do git com o comando git rm;
Apos deletar um arquivo do git, ele não terá mais suas ataulizações consideradas pelo git;
Apenas quando for adicionado novamente com o git add.

-git log PC: log
Podemos acessar um log de modificações feitas no projeto com o git log;
Voce receberá um informação dos commits realizados no projeto até então.

-git mv (nome do arquivo antigo) (nome do arquivo novo ou diretorio pasta nova) PC: mover, renomear
Com o git mv podemos renomear arquivos e mover arquivos de pastas;
E isso fará com que este novo arquivo seja monitoriado pelo git;
O arquivo anterior sera excluido.

-git checkout (nome do arquivo) PC: desfazer
O arquivo modificado pode ser retornado ao estado original (estado que o arquivo esta no repositório);
Apos a utilização do mesmo, o arquivo sai do staging (git add);
Caso seja feita uma proxima alteração, ele entra em staging novamente.
Atenção: O arquivo modificado será perdido, ou seja, as alterações feitas serão descartadas.

git reset --hard origin/main (Volta a versão que esta no repositório)PC: desfazer
Todas as alterações commitadas e também as pendentes seão excluidas;
Ao contrario do git checkout que desfaz apenas o arquivo do staging, o git reset --hard desfaz tudo, inclusive commits;

- .gitignore PC: ignorar
Devemos criar um arquivo chamado .gitignore para ignorar arquivos no git;
Uma tecnica muito utilizada é ignorar arquivos do projeto;
Devemos inserir um arquivo chamado .gitignore na raiz do projeto;
Nele podemos inserir todos os arquivos qeu não devem entrar no versionamento;
Isso é util para arquivos gerados automaticamente ou arquivos que contêm informações sensíveis (senhas, chaves de API, etc).






MACETES

 - M1 - (O git usa o Vim como padrão)
Se você der um commit -a e não colocar uma mensagem, aparecera algo como:

# Please enter the commit message for your changes. Lines starting
# with '#' will be ignored, and an empty message aborts the commit.
#
# On branch master
# Changes to be committed:
#       new file:   CSS/style.css
#       new file:   b.txt
#       new file:   index.html
#       new file:   teste.txt
#
~   

O que fazer?
# será ignorado pelo git
~ indica linhas vazias
1 - Pressione a tecla 'i'; - Coloca o Vim em modo de edição de texto. (Vim é um editor de texto dentro do terminal)
Você deve escrever acima do primeiro #;

2 - Digite a mensagem do commit

3 - Pressione ESC - Sai do modo de edição;

4 - Digite :wq - O 'w' salva(write) e o 'q' sai(quit);

5- commit finalizado;

Ou

Para cancelar o commit:
1 - Pressione ESC;
2- Digite :q!
3 - Enter.
